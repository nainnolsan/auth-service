## GET /api/auth/profile

Devuelve los datos del usuario correspondiente al access token proporcionado. Es una ruta **protegida** — requiere autenticación.

## Autenticación requerida

Incluye el access token en el header `Authorization`:

```http
Authorization: Bearer <accessToken>
```

<Warning>
  Si el access token ha expirado, recibirás un `401`. Usa el endpoint [/refresh](/auth-service/refresh) para obtener uno nuevo antes de reintentar.
</Warning>

## Ejemplos

<CodeGroup>

```bash cURL
curl -X GET https://tu-auth-service.railway.app/api/auth/profile \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

```js JavaScript (fetch)
const response = await fetch('/api/auth/profile', {
  method: 'GET',
  headers: {
    'Authorization': `Bearer ${localStorage.getItem('accessToken')}`
  }
});

const data = await response.json();
```

```graphql GraphQL (via API Gateway)
query Me {
  me {
    id
    name
    email
  }
}
```

</CodeGroup>
