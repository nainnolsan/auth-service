## ¿Qué es el Auth Service?

El **Auth Service** es un microservicio independiente que maneja todo lo relacionado con usuarios: registro, inicio de sesión, renovación de tokens y cierre de sesión. Utiliza **JWT** (JSON Web Tokens) para autenticación stateless y **PostgreSQL** para persistir los datos.

## Base URL

<Note>
  En producción el API Gateway consume este servicio internamente. Los endpoints listados aquí son los del servicio directo.
</Note>

| Entorno     | URL                                           |
|-------------|-----------------------------------------------|
| Local       | `http://localhost:3001`                       |
| Producción  | Servicio interno en Railway (no expuesto)     |

## Sistema de Tokens JWT

El servicio utiliza un sistema de **doble token** para balancear seguridad y experiencia de usuario.

| Token         | Duración | Propósito                                          |
|---------------|----------|----------------------------------------------------|
| Access Token  | 15 min   | Autenticar cada request a rutas protegidas         |
| Refresh Token | 7 días   | Obtener un nuevo access token sin volver a loguearse |

El **Refresh Token** se almacena en la base de datos, lo que permite invalidarlo explícitamente al hacer logout.

## Modelo de Usuario

La tabla `users` en PostgreSQL tiene la siguiente estructura:

| Campo          | Tipo        | Descripción                          |
|----------------|-------------|--------------------------------------|
| `id`           | UUID        | Identificador único autogenerado     |
| `name`         | varchar(50) | Nombre del usuario                   |
| `email`        | varchar(255)| Email único                          |
| `password`     | varchar(255)| Contraseña hasheada con bcrypt       |
| `refreshTokens`| text[]      | Array de refresh tokens activos      |
| `createdAt`    | timestamp   | Fecha de creación                    |
| `updatedAt`    | timestamp   | Fecha de última actualización        |

<Warning>
  La contraseña **nunca** se devuelve en ninguna respuesta de la API. El modelo omite automáticamente `password` y `refreshTokens` al serializar.
</Warning>

## Autenticación en rutas protegidas

Para acceder a rutas protegidas (como `/profile`), incluye el access token en el header:

```http
Authorization: Bearer <accessToken>
```

## Variables de entorno requeridas

```env
PORT=3001
DATABASE_URL=postgresql://user:password@host:5432/db
JWT_SECRET=tu_secreto_super_seguro
JWT_REFRESH_SECRET=tu_refresh_secreto_super_seguro
JWT_EXPIRES_IN=15m
JWT_REFRESH_EXPIRES_IN=7d
CORS_ORIGIN=http://localhost:5173
```
